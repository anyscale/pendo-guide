You can launch a hosted VSCode with 1-click in your browser or on your local machine that connects to the Workspace.
Workspace provides handy tools for writing your Ray code on the head node of the underlying cluster. Specify additional dependencies via runtime environment.
Everytime you run your Ray code, the head node sends the Ray code and the runtime env to every worker node to be installed.
Cluster environments are specially-built Docker images that work efficiently with Ray inside of Anyscale. Learn more.
/mnt/cluster_storage This is the NFS of the cluster and persisted with the workspace.
/mnt/user_storage This is a NFS only accessible to the Anyscale user (from their workspaces, jobs, and services).
/mnt/shared_storage This is a NFS accessible to all the Anyscale users of the same Anyscale Cloud.
/home/ray/ This is where your workspace files are placed (default directory of the webterminal). This storage is very fast, but note that we only persist non-gitignored files here. Files outside this directory will not be persisted.
When creating a new Workspace, use a compute config to specify the node types and numbers for the underlying cluster.
When new worker nodes are added to the underlying Ray cluster, Anyscale mounts the storage volumes and installs the cluster env to each of them.
You can store environment variables and credentials (e.g. wandb API key) associated with your user account by running these commands from the web terminal within the workspace:
  • python -m anyscale.snapshot_util put_env
  • python -m anyscale.snapshot_util get_env
  • python -m anyscale.snapshot_util del_env
Your unique SSH key tied is available in ~/.ssh/id_rsa in your workspace cluster. Authorize this generated key to access GitHub or other services.
We support laptop-centric users via the "anyscale workspace" group of CLI commands. (Note: the workspace cluster must be running for these commands to be used):
  • anyscale workspace clone -n
From within a locally cloned workspace directory:
  • anyscale workspace pull
  • anyscale workspace push
  • anyscale workspace run 'python main.py' [--no-push] [--as-job]
